<!DOCTYPE html>

Define que este documento es un documento HTML5, ayuda a los navegadores a mostrar las páginas web correctamente.
Solo debe aparecer una vez, en la parte superior (antes que cualquier etiqueta HTML).

<html>

Es el elemento raíz de una página HTML. El documento html inicia con la etiqueta <html> y termina con </html>.
El atributo lang en la etiqueta de apertura es el único atributo que necesitamos especificar en HTML5. Este atributo define el idioma humano del contenido del documento que estamos creando.

<body>

La parte visible de un documento HTML se encuentra entre la etiquetas <body> y </body>.

<head>

Dentro de las etiquetas <head> y </head> definiremos el título de nuestra página web, declararemos el set de caracteres correspondiente, proveeremos información general acerca del documento e incorporaremos los archivos externos con estilos, códigos Javascript o incluso imágenes necesarias para generar la página en la pantalla. Excepto por el título y algunos íconos, el resto de la información incorporada en el documento entre estas etiquetas es invisible para el usuario.

<meta>

Junto al atributo charset seguido de la codificacion define el juego de caracteres a utilizar para mostrar el documento.
Se utiliza como se muestra a continuacion.
<meta charset="UTF-8">.
La etiqueta <meta> acepta otros atributos como name o content.

<title>

El texto entre las etiquetas <title> es el título del documento que estamos creando. Normalmente este texto es mostrado en la barra superior de la ventana del navegador.

<link>

Este elemento es usado para incorporar estilos, códigos Javascript, imágenes o iconos desde archivos externos.
Aqui un ejemplo de como agregar un documento de estilo css.
<link rel=”stylesheet” href=”misestilos.css”>
En HTML5 Solo necesitamos dos atributos para incorporar nuestro archivo de estilos: rel y href. El atributo rel significa “relación” y es acerca de la relación entre el documento y el archivo que estamos incorporando por medio de href.

<head> <link rel="stylesheet" href="styles.css"> </head>

<header>

Representa un contenedor para contenido introductorio o un conjunto de enlaces de navegacion.
Admite atributos globales y de evento.
Normalmente contiene:
  • Uno o más elementos de encabezado (<h1> - <h6>)
  • Logotipo o icono.
  • Informacion de autoría.

<main>

Especifica el contenido principal de un documento, debe ser exclusivo del documento. No debe tener contenido repetido en otros documentos, como barras laterales, enlaces de navegacion, información de autor, Logotipos del sitio y formulario de búsqueda.
No debe haber más de uno, ni debe ser descendiente de un elemento <aside>, <article>, <footer>, <header> o <nav>.
Admite atributos globales y de evento.

<article>

Especifica contenido independiente y autonomo. Un articúlo debe tener sentido por sí mismo y debe ser posible distribuirlo independientemente del resto del site. No se visualiza de forma especial en el navegador.
Fuentes potenciales de <article>,
  • Publicación en Foro.
  • Entrada de Blog.
  • Noticia

Admite atributos globales y de evento.

Ejemplo de uso

<article>
<h2>Microsoft Edge</h2>
<p>Microsoft Edge is a web browser developed by Microsoft, released in 2015. Microsoft Edge replaced Internet Explorer.</p>
</article>

<aside>

Define algún contenido además del contenido en el que se coloca. El contenido adicional debe estar relacionado indirectamente con el contenido circundante.
Consejo: El contenido <aside> a menudo se coloca como una barra lateral en un documento.
Admite atributos globales y de evento.

Ejemplo de uso

<aside>
<p>The Epcot center is a theme park at Walt Disney World Resort featuring exciting attractions, international pavilions, award-winning fireworks and seasonal special events.</p>
</aside>

<nav>

Define un conjunto de enlaces de navegación.
Tenga en cuenta que NO todos los enlaces de un documento deben estar dentro de un elemento <nav>. Este elemento está diseñado únicamente para bloques principales de enlaces de navegación.
Los navegadores, como los lectores de pantalla para usuarios discapacitados, pueden usar este elemento para determinar si se debe omitir la representación inicial de este contenido.
Admite atributos globales y de evento.

Ejemplo de uso

<nav>
<a href="/html/">HTML</a>
<a href="/css/">CSS</a>
<a href="/js/">JavaScript</a>
<a href="/python/">Python</a>
</nav>

<footer>

Define un pie de página para un documento o sección. Un <footer>
normalmente contiene:
  • información de derechos de autor
  • Información del contacto
  • mapa del sitio
  • volver a los enlaces superiores
  • documentos relacionados

Admite atributos globales y de evento.

Ejemplo de uso

<footer>
<p>Author: Hege Refsnes</p>
<p><a href="mailto:hege@example.com">hege@example.com</a></p>
</footer>

<div>

Define una división o una sección en un documento HTML. La etiqueta <div> e puede diseñar fácilmente mediante el atributo de clase o id.

De forma predeterminada, los navegadores siempre colocan un salto de línea antes y después del <div>
Admite atributos globales y de evento.

Ejemplo de uso

<div class="myDiv">
<h2>This is a heading in a div element</h2>
<p>This is some text in a div element.</p>
</div>

<a>

Define un hipervínculo, que se utiliza para vincular de una página a otra.
El atributo más importante del elemento <a> es el atributo href, que indica el destino del enlace.
De forma predeterminada, los enlaces aparecerán de la siguiente manera en todos los navegadores:
  • Un enlace no visitado esta subrayado y azul.
  • Un enlace visitado esta subrayado y en color morado.
  • Un enlace activo esta subrayado y n rojo

Admite atributos globales y de evento.

Ejemplo de uso

<a href="https://www.w3schools.com">Visit W3Schools.com!</a>
Visit W3Schools.com!

<abbr>

Define una abreviatura o un acrónimo, como "HTML", "CSS", "Sr.", "Dr.", "ASAP", "ATM".
Consejo: utilice el atributo global para mostrar la descripción de la abreviatura/acrónimo cuando pase el mouse sobre el atributo title.
Admite atributos globales y de evento.

Ejemplo de uso

The <abbr title="World Health Organization">WHO</abbr> was founded in 1948.
The WHO was founded in 1948.

La información de contacto puede ser una dirección de correo electrónico, URL, dirección física, número de teléfono, nombre de usuario en redes sociales, etc.
El texto del normalmente se muestra en cursiva y los navegadores siempre agregarán un salto de línea antes y después del elemento.

Admite atributos globales y de evento.

Ejemplo de uso

<audio>

La etiqueta <audio> se utiliza para incrustar contenido de sonido en un documento, como música u otras transmisiones de audio.
La <audio>etiqueta contiene una o más etiquetas <source> con diferentes fuentes de audio. El navegador seleccionará la primera fuente compatible.
El texto entre las etiquetas <audio>y </audio> solo se mostrará en navegadores que no admitan el elemento <audio>.
Hay tres formatos de audio admitidos en HTML: MP3, WAV y OGG.
Admite atributos globales y de evento.

Ejemplo de uso

<audio controls>
<source src="horse.ogg" type="audio/ogg">
<source src="horse.mp3" type="audio/mpeg">
Your browser does not support the audio tag.
</audio>

Admite atributos globales y de evento.

Ejemplo de uso

This is normal text - and this is bold text.

<base>

La etiqueta <base> especifica la URL base y/o el destino de todas las URL relativas en un documento.
La etiqueta <base> debe tener presente un atributo href o un atributo target, o ambos.
Sólo puede haber un único elemento <base> en un documento, y debe estar dentro del elemento <head>.

Admite atributos globales pero no de evento.

Ejemplo de uso

<head>
<base href="https://www.w3schools.com/" target="_blank">
</head>
<body>
<img src="images/stickman.gif" width="24" height="39" alt="Stickman">
<a href="tags/tag_base.asp">HTML base Tag</a>
</body>

<button>

La etiqueta <button> define un botón en el que se puede hacer clic.
Dentro de un elemento <button> se puede colocar texto (y etiquetas como <i>, <b>, <strong>, <br>, <img>, etc.). Esto no es posible con un botón creado con el elemento <input>.
Consejo: especifique siempre el atributo type de un elemento <button> para indicar a los navegadores qué tipo de botón es.
Admite atributos globales y de evento.

Ejemplo de uso

<button type="button">Click Me!</button>

    El uso más común de este <picture>elemento será la dirección artística en diseños adaptables. En lugar de tener una imagen que se amplía o reduce según el ancho de la ventana gráfica, se pueden diseñar varias imágenes para que ocupen mejor la ventana gráfica del navegador.

    <script>

    La <script>etiqueta se utiliza para incrustar un script del lado del cliente (JavaScript).
    El <script>elemento contiene declaraciones de script o apunta a un archivo de script externo a través del atributo src.
    Los usos comunes de JavaScript son la manipulación de imágenes, la validación de formularios y los cambios dinámicos de contenido.
    Consejo: También observe el elemento <noscript> para los usuarios que tienen scripts deshabilitados en su navegador o que tienen un navegador que no admite scripts del lado del cliente.

    Admite atributos globales y de evento.

    Ejemplo de uso

    <script>
    document.getElementById("demo").innerHTML = "Hello JavaScript!";
    </script>

    <style>

    Dentro del <style>elemento se especifica cómo deben representarse los elementos HTML en un navegador.
    El <style>elemento debe estar incluido dentro de la <head>sección del documento.
    Nota: Cuando un navegador lee una hoja de estilo, formatea el documento HTML según la información contenida en ella. Si se han definido propiedades para el mismo selector (elemento) en diferentes hojas de estilo, se utilizará el valor de la última hoja de estilo leída.

    Admite atributos globales y de evento.

    <p>A paragraph.</p>
    </body>
    </html>

    <tbody>

    <template>

    La <template>etiqueta se utiliza como contenedor para guardar cierto contenido HTML oculto para el usuario cuando se carga la página.
    El contenido interior <template>se puede representar más tarde con JavaScript.
    Puedes usar la <template>etiqueta si tienes código HTML que quieras usar repetidamente, pero solo si lo solicitas.
    Para hacerlo sin la <template>etiqueta, debes crear el código HTML con JavaScript para evitar que el navegador lo renderice.

    Admite atributos globales.

    Ejemplo de uso

    <button onclick="showContent()">Show hidden content</button>
    <template>
    <h2>Flower</h2>
    <img src="img_white_flower.jpg" width="214" height="204">
    </template>
    <script>
    function showContent() {
    let temp = document.getElementsByTagName("template")[0];
    let clon = temp.content.cloneNode(true);
    document.body.appendChild(clon);
    }
    </script>
